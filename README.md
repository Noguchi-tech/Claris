# Claris / クラリス

自分用のタスク、メモ、方針管理アプリです。まずは iPhone の Safari でホーム画面に追加して使える、ローカル保存の PWA として実装しています。

## 起動

```powershell
node server.mjs
```

起動後に `http://127.0.0.1:4173/` を開きます。

## 主な機能

- 下部タブ中央の「今日」
- カレンダー、今日、一覧の3タブ構成
- タスク、メモ、方針を同じ一覧画面で管理
- 一覧画面内の走り書きメモと録音
- 追加ボタンからタスク、メモ、方針を選択して追加
- タスクごとの担当者設定
- 実施日と `DL` の分離
- 実施日選択時の優先度空き状況表示
- 週次、毎月日付、第n曜日、月末、カスタム日付の繰り返しタスク
- タスク一覧の実施日順ソート
- タスク削除前の確認
- 削除済みタスク、メモ、方針の保管と復元
- 優先度枠の重複時に、既存タスクの移動先を選べる入れ替え UI
- 月曜始まりカレンダー、分類/PJフィルター、DL/方針・施策の簡易表示
- メモタイトルと本文プレビューからの編集、タスク側からの関連メモ確認
- 本文と文字起こしから議題、決定事項、ネクストアクションを自動判定
- 録音メモの端末内保存と専用欄への文字起こし保存
- JSON / CSV / テキスト貼り付けインポート
- JSON エクスポート
- IndexedDB によるローカル保存

## 公開URLとマスタデータ

GitHub Pages の公開URLは静的配信なので、アプリから安全にマスタファイルへ直接書き戻すことはしません。直接書き込みを公開ページに持たせると、認証情報や書き込み口の扱いが難しくなります。

当面の運用は、設定の「マスター上書き用JSON」でiPhone側の最新状態を書き出し、PC側で `data/claris-master-2026-05-17.json` の更新に使う形です。アプリ起動時にこのJSONを取得し、`fullSync: true` かつ未適用の `importId` であれば、この端末のIndexedDBへ上書き同期します。パスキーなどで安全に双方向同期する場合は、GitHub Pagesだけで完結させず、認証と書き込み権限を持つバックエンドを別途用意します。
