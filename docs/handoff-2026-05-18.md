## 実施できなかった項目

### アプリ終了後も継続する LLM 処理

静的 PWA 単体では、iPhone でページ移動、アプリ切り替え、ホーム画面 PWA の終了後も JavaScript 処理や fetch の結果待ちを保証できない。2026-05-20 時点の標準仕様では、自動判定中の UI、外部 LLM エンドポイント連携、失敗時のローカル判定フォールバックは廃止済みであり、Claris 本体は AI 整理用 JSON ファイル共有と整理結果 JSON の取り込み検証だけを行う。

アプリを落としても処理を続けるには、次のいずれかが必要。

- 外部バックエンドでジョブを受け付け、完了結果を次回起動時に取得する。
- Power Automate、Cloudflare Workers、Vercel Functions などに LLM 判定 API を置く。
- iOS ネイティブアプリまたはラッパーアプリで Background Tasks を扱う。

### Safari 実機でのホーム画面 PWA 検証

ローカルのブラウザ幅検証は可能だが、実機 iPhone16 のホーム画面追加 PWA と Safari 音声認識の権限挙動は端末上で最終確認する必要がある。特に Web Speech API は iOS のバージョン、Safari/PWA の実行形態、マイク権限で挙動が変わる。

## 推奨方針

現在の PWA は、タスク、メモ、運営情報のローカル管理には適している。今後 AI 整理を自動実行する業務導線が必要になっても、現時点の Claris 本体へ直接 POST 処理を戻さず、標準4ファイルで理由、影響範囲、移行手順を整理してから別途判断する。

Windows との文書同期は、`Claris_app/docs/` を GitHub 管理下に置く現在の構成が適切。GitHub に push すれば Windows 側で pull して同じ要件定義書、設計書、仕様書を参照できる。
