# Claris 要件定義書

更新日: 2026-05-18  
対象: iPhone16 の Safari でホーム画面に追加して使う静的 PWA

## 1. 目的

Claris は、個人業務のタスク、メモ、運営情報を iPhone 上で素早く整理するローカル保存型 PWA である。主対象は iPhone16 の縦画面で、今日の実施、DL、優先度、期間を持つ運営情報を直感的に確認・更新できる状態を維持する。

## 2. 利用環境

- 主利用環境は iPhone16 のホーム画面追加 PWA。
- データは IndexedDB `claris-local-db` に保存する。
- 静的配信で動作し、Service Worker によりオフラインキャッシュを持つ。
- PC ブラウザは編集・検証・GitHub 経由のファイル更新に使う。

## 3. UI 要件

- 下部タブは「カレンダー」「今日」「一覧」の3タブとする。
- 下部タブのリキッドグラス風インジケーターはタブ枠内に収め、アクティブタブと連動して横移動する。
- 今日タブでは、最優先、2次優先、3次優先をサブタスクと視覚的に分ける。
- 入力フォーム下部の保存領域は、入力カードの面積を保ちながら縦幅を抑える。
- 操作完了時は toast またはインラインアニメーションで反応を返す。

## 4. タスク要件

- タスクの優先度は、最優先、2次優先、3次優先、サブタスクのいずれかに必ず分類する。
- 「未選択」は仕様から廃止する。
- タスクフォームでは、タスク側から関連メモを検索し、タイトル、本文、文字起こしを対象に絞り込める。
- 優先度枠の重複時は既存の入れ替え UI を維持する。

## 5. メモ要件

- メモ整理欄の表記は、アプリ全体で「論点」「方針」「行動」に統一する。
- 自動判定はタイトル、本文、文字起こしを入力元にする。
- 一覧タブの走り書きメモでは、本文を追加して保存した場合も録音中の文字起こしをメモの文字起こし欄へ保存する。
- 録音と文字起こしの保存前状態は画面上のプレビューと内部ドラフトで保持する。

## 6. 運営情報要件

- 運営情報はタイトル、種別、期間、分類、内容を持つ。
- 種別は設定画面で編集でき、運営情報フォームの種別選択からも新規追加できる。
- カレンダー上では同じ日に同じ種別が複数ある場合、個別に並べず `半2` のように種別略称と件数でまとめる。
- 期間設定では、期間保存後に反応を返し、左下表示を `5/18-6/30` のような範囲表示にする。

## 7. 設定要件

- 分類と運営情報の種別は設定画面で編集できる。
- 分類と種別の順番入れ替えは矢印ボタンではなく、ドラッグ操作で行う。
- 削除済みデータの保管と復元は維持する。

## 8. LLM 連携要件

- 設定に連携名とエンドポイントを持ち、自動判定ボタンから外部 LLM へ POST できる。
- 外部連携失敗時はローカル判定へ戻す。
- 静的 PWA だけでは、アプリ終了後も処理を継続するバックグラウンド LLM 実行は保証しない。必要な場合はバックエンドまたはネイティブ実装を追加する。

## 9. ドキュメント要件

- 要件定義書、設計書、仕様書は `Claris_app/docs/` に格納する。
- `Claris_app` は Git 管理下のため、GitHub 経由で Windows 環境へ pull する運用に適する。
